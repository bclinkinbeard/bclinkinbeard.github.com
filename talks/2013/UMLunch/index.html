Browserify

Browserify

  • Client-side JavaScript valhalla
  • Buh-bye script tag soup and RequireJS

CommonJS style modules

  • What Node.js uses
  • Synchronous, which is where Browserify comes in
  • ...but first

CommonJS style modules

  • Module returns what 
    module.exports
    holds
  • Can be any type. Function, object, string, number...
  • module.exports = {
	name: 'Ben',
	age: 33,
	writeCode: function(){...}
}

CommonJS style modules

  • Can add to 
    exports
    directly instead
  • exports.name = 'Ben';
  • exports.age = 33;
  • exports.writeCode = function(){...};

CommonJS style modules

  • Load other modules with 
    require()
  • require('path')
    looks in 
    node_modules/node core API
  • require('./someFile')
    uses relative file path
  • Assign 
    var someModule = require('./someFile');
  • Or just use 
    require('path').resolve('./../a/crazy/path');

Browserify

  • Write client side code using CommonJS style modules
  • Specify one or more 'entry' files
  • Browserify walks 
    require()
    statements and concatenates files
  • Each file's contents wrapped in a closure
  • Supports source maps and all kinds of other goodies

Browserify

var Class = require( 'Class' );

var AppController = module.exports = Class.extend( {

	init: function( $scope ) {
		this.$scope = $scope;
	}

} );

AppController.$inject = [ '$scope' ];
