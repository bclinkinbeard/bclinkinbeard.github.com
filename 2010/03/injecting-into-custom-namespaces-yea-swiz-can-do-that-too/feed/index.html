Swiz now supports custom namespaces out of the box, on all metadata tags. [Inject], [Mediate], [PostConstruct], etc can all be defined on custom namespaced members. The only caveat is that if you are using bean property injection like [Inject( "someBean.someProp" )], you need to add bind=false to the tag if someProp is bindable. By default, Swiz will create a binding for you when injecting a bindable bean property. However, BindingUtils only works with public members, so we can't do that in that scenario.
